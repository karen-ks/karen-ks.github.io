Analysing Feature Contributions of Model Outputs: Credit Card Default & Phishing Dataset

A reflective report of the analysis of feature contributions on the basis of the statistical distribution of the model's features.

17 December 2021

Complete PDF
Github Repo

Introduction

The project is based on the exploration of the statistical distribution of dataset features based on their feature importance and classification results. The objective was to explore the attributes with the highest feature importance and determine whether its contribution is relevant to the model's output.

Methodology

Kaggle and the UCI Repository were used as sources for the chosen Credit Card Default and Phishing Detection datasets. The selected Machine Learning algorithms were Decision Trees, Random Forests, Gradient Boosting and the XGBoost algorithms.

After data cleaning and feature engineering, three adjusted datasets were prepared as follows:

  • Principal Component Analysis applied
  • Inclduing only 50% of positive entries
  • Inclduing only 20% of positive entries

    • A blind run was carried out on each dataset to determine baseline results. GridSearchCV was utilised for hyperparameter tuning using both 'F1 Score' and 'Accuracy'scoring. Visualisation was carried out next for the feature importance of each dataset (and their adjusted versions). This included both the classifier's feature importance attribute and the Mutual Information scoring method to check for inconsistencies.

    Results & Discussion

    High accuracy and F1 scores were noted on the original datasets. As for the adjusted datasets with 50%/20% of the original positive input entries, they also indicated as high a scoring as the original dataset. The feature with the highest impact was consisten across the adjusted datasets.

    Future Work

    The future aim is to expand this project to map specific inputs with stronger influences on the output of the model to visualise and explain their relationships.
